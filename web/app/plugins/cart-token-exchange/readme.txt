Handles cart token restoration from Louis search application to WooCommerce store.

== Description ==

The Cart Token Exchange plugin enables seamless cart restoration when users navigate from the Louis search application (louis.aph.org) to the WooCommerce store (aph.org). 

**Key Features:**

* JWT token validation for secure cart restoration
* Automatic cart restoration from Louis application
* Rate limiting to prevent abuse
* Debug logging for troubleshooting
* Admin settings for configuration
* Session management integration
* Error handling with graceful fallback

**How it works:**

1. User searches and adds items to cart in Louis application
2. Louis generates a cart token using WooCommerce Store API
3. User navigates to checkout with cart token parameter
4. Plugin validates token and restores cart contents
5. User proceeds with normal WooCommerce checkout

== Installation ==

1. Upload the plugin files to `/wp-content/plugins/cart-token-exchange/`
2. Activate the plugin through the 'Plugins' screen in WordPress
3. Navigate to WooCommerce > Settings > Cart Token Exchange
4. Configure settings as needed

== Frequently Asked Questions ==

= What is a cart token? =

A cart token is a JWT (JSON Web Token) generated by the WooCommerce Store API that contains cart session information for headless commerce applications.

= How long are cart tokens valid? =

Cart tokens are typically valid for 48 hours by default, but this can be configured in the plugin settings.

= What happens if the token is invalid? =

The plugin will fail silently and allow the user to continue with an empty cart. Error details are logged if debug logging is enabled.
