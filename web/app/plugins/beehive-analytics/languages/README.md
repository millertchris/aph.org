# Translation POT File Structure

This plugin uses a modular approach to translation files, with separate POT files for each package that are then merged into a single master POT file.

## Package POT Files

Each package generates its own POT file during the build process:

### 1. **admin.js.pot**

- Location: `languages/admin.js.pot`
- Source: `packages/admin/src/**/*.{ts,tsx}`
- Contains: All translations from the admin interface
- Pages: Settings, Accounts, Dashboard, Google Analytics, GTM, Onboarding

### 2. **common.js.pot**

- Location: `languages/common.js.pot`
- Source: `packages/common/**/*.{ts,tsx}`
- Contains: Shared components and utilities
- Components: Charts, Google Authentication, Select Accounts, Loading, etc.

### 3. **blocks.js.pot**

- Location: `languages/blocks.js.pot`
- Source: `packages/blocks/src/**/*.{ts,tsx}`
- Contains: Gutenberg blocks translations

### 4. **popular-widget.js.pot**

- Location: `languages/popular-widget.js.pot`
- Source: `packages/popular-widget/src/**/*.{ts,tsx}`
- Contains: Popular posts widget translations

### 5. **visitors-widget.js.pot**

- Location: `languages/visitors-widget.js.pot`
- Source: `packages/visitors-widget/src/**/*.{ts,tsx}`
- Contains: Visitors widget translations
- Tabs: General Stats, Audience, Top Pages & Views, Traffic

## Master POT File

### **ga_trans.pot**

- Location: `languages/ga_trans.pot`
- Contains: All PHP translations + merged JavaScript translations from all packages
- Generated by: `pnpm run plugin-pot`

## Text Domain

All translations use the text domain: **`ga_trans`**

### PHP Usage

```php
__( 'Text to translate', 'ga_trans' )
esc_html__( 'Text to translate', 'ga_trans' )
```

### JavaScript/TypeScript Usage

```typescript
import { __ } from '@wordpress/i18n'

__('Text to translate', 'ga_trans')
```

## Generating POT Files

### Development Workflow

1. **Build packages** (generates individual package POT files):

    ```bash
    pnpm run build
    ```

2. **Generate master POT file** (merges all POT files):
    ```bash
    pnpm run plugin-pot
    ```

### What Happens During Build

- Webpack processes each package's TypeScript/JavaScript files
- `@wordpress/babel-plugin-makepot` extracts translation strings
- Individual POT files are created in `languages/` folder

### What Happens During plugin-pot

- WP-CLI scans all PHP files for translation strings
- Merges all 5 JavaScript POT files
- Creates final `ga_trans.pot` with all translations

## File Structure

```
languages/
├── admin.js.pot              # Admin package translations
├── common.js.pot             # Common package translations
├── blocks.js.pot             # Blocks package translations
├── popular-widget.js.pot     # Popular widget translations
├── visitors-widget.js.pot    # Visitors widget translations
└── ga_trans.pot              # Master POT file (PHP + all JS merged)
```

## Notes

- Package POT files are kept separate for easier maintenance
- Translators only need to work with `ga_trans.pot`
- All translation functions must include the `ga_trans` text domain
- POT files should be regenerated after adding new translatable strings
